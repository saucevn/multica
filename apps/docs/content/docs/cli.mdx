The Hira CLI mirrors almost everything the Web UI can do (create [issues](/issues), assign [agents](/agents), start the [daemon](/daemon-runtimes), and more). This page lists every top-level command with a one-line description. For the full set of flags and examples, run `hira <command> --help`.

## Getting authenticated

Run this the first time you use the CLI to obtain a **personal access token (PAT)**:

```bash
hira login
```

Your browser opens automatically. After you approve in the web app, the CLI saves the PAT (prefixed with `mul_`) to `~/.hira/config.json`. Every subsequent command authenticates with that PAT.

<Callout type="tip">
For CI or headless environments, skip the browser flow: create a PAT in the web app under **Settings → Personal Access Tokens**, then run `hira login --token <mul_...>` to supply it directly.
</Callout>

For the difference between token types, see [Authentication and tokens](/auth-tokens).

## Auth and setup

| Command | Purpose |
|---|---|
| `hira login` | Log in and save a PAT |
| `hira auth status` | Show current login status, user, and workspace |
| `hira auth logout` | Clear the local PAT |
| `hira setup cloud` | One-shot setup for Hira Cloud (login + install daemon) |
| `hira setup self-host` | One-shot setup for a self-hosted backend |

## Workspaces and members

| Command | Purpose |
|---|---|
| `hira workspace list` | List every workspace you can access |
| `hira workspace get <slug>` | Show details for one workspace |
| `hira workspace members` | List members of the current workspace |

## Issues and projects

| Command | Purpose |
|---|---|
| `hira issue list` | List issues |
| `hira issue get <id>` | Show a single issue |
| `hira issue create --title "..."` | Create a new issue |
| `hira issue update <id> ...` | Update an issue (status, priority, assignee, etc.) |
| `hira issue assign <id> --agent <slug>` | Assign to an agent (triggers a task immediately) |
| `hira issue status <id> --set <status>` | Shortcut to change status |
| `hira issue search <query>` | Keyword search |
| `hira issue runs <id>` | Show agent runs on an issue |
| `hira issue rerun <id>` | Rerun the most recent agent task |
| `hira issue comment <id> ...` | Nested: view / post comments |
| `hira issue subscriber <id> ...` | Nested: subscribe / unsubscribe |
| `hira project list/get/create/update/delete/status` | Project CRUD |

## Agents and skills

| Command | Purpose |
|---|---|
| `hira agent list` | List the workspace's agents |
| `hira agent get <slug>` | Show an agent's configuration |
| `hira agent create ...` | Create an agent |
| `hira agent update <slug> ...` | Update an agent |
| `hira agent archive <slug>` | Archive |
| `hira agent restore <slug>` | Restore an archived agent |
| `hira agent tasks <slug>` | Show an agent's task history |
| `hira agent skills ...` | Nested: attach / detach skills |
| `hira skill list/get/create/update/delete` | Skill CRUD |
| `hira skill import ...` | Import a skill from GitHub, ClawHub, or the local machine |
| `hira skill files ...` | Nested: manage a skill's files |

## Autopilots

| Command | Purpose |
|---|---|
| `hira autopilot list` | List every autopilot in the workspace |
| `hira autopilot get <id>` | Show a single autopilot |
| `hira autopilot create ...` | Create an autopilot |
| `hira autopilot update <id> ...` | Update |
| `hira autopilot delete <id>` | Delete |
| `hira autopilot runs <id>` | Show run history |
| `hira autopilot trigger <id>` | Trigger a run manually |

## Daemon and runtimes

| Command | Purpose |
|---|---|
| `hira daemon start` | Start the daemon (background by default; add `--foreground` to run in the foreground) |
| `hira daemon stop` | Stop the daemon |
| `hira daemon restart` | Restart the daemon |
| `hira daemon status` | Check whether the daemon is online and its concurrency |
| `hira daemon logs` | View daemon logs |
| `hira runtime list` | List runtimes in the current workspace |
| `hira runtime usage` | Show resource usage |
| `hira runtime activity` | Recent activity log |
| `hira runtime ping <id>` | Ping a runtime to check it's online |
| `hira runtime update <id> ...` | Update a runtime's configuration |

## Miscellaneous

| Command | Purpose |
|---|---|
| `hira repo checkout <url>` | Clone a repo locally for agents to use |
| `hira config` | View or edit local CLI configuration |
| `hira version` | Print the CLI version |
| `hira update` | Upgrade the CLI to the latest release |
| `hira attachment download <id>` | Download an attachment from an issue or comment |

## Getting full flags

Every command supports `--help`:

```bash
hira issue create --help
hira agent update --help
```

v2 will ship a dedicated detailed reference page for each command.
