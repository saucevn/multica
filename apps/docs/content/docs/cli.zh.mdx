Hira CLI 把 Web UI 能做的事几乎全部搬到了命令行上（创建 [issue](/issues)、分配 [智能体](/agents)、启动 [守护进程](/daemon-runtimes) 等等）。这一页把所有顶级命令列出来，每条配一句用途。完整 flag 和示例用 `hira <命令> --help` 查。

## 认证入口

第一次用 CLI 时先登录，拿一个**个人访问令牌（Personal Access Token，PAT）**：

```bash
hira login
```

浏览器会自动打开，你在 Web 端同意后，CLI 把 PAT（`mul_` 前缀）保存到 `~/.hira/config.json`。此后所有命令都会自动用这个 PAT 认证。

<Callout type="tip">
CI / 无浏览器环境跳过浏览器流程：先在 Web 端 **Settings → Personal Access Tokens** 创建一个 PAT，然后 `hira login --token <mul_...>` 直接填入。
</Callout>

Token 类型的详细区分见 [认证与令牌](/auth-tokens)。

## 认证与初始化

| 命令 | 用途 |
|---|---|
| `hira login` | 登录并保存 PAT |
| `hira auth status` | 查看当前登录状态、用户、工作区 |
| `hira auth logout` | 清除本地 PAT |
| `hira setup cloud` | Hira Cloud 一键初始化（登录 + 装 daemon） |
| `hira setup self-host` | 自部署后端的一键初始化 |

## 工作区和成员

| 命令 | 用途 |
|---|---|
| `hira workspace list` | 列出你有权访问的所有工作区 |
| `hira workspace get <slug>` | 查看一个工作区的详情 |
| `hira workspace members` | 列出当前工作区的成员 |

## Issue 和 Project

| 命令 | 用途 |
|---|---|
| `hira issue list` | 列出 issue |
| `hira issue get <id>` | 查看单条 issue |
| `hira issue create --title "..."` | 创建新 issue |
| `hira issue update <id> ...` | 修改 issue（状态、优先级、分配人等） |
| `hira issue assign <id> --agent <slug>` | 分配给智能体（立即触发任务） |
| `hira issue status <id> --set <status>` | 快捷改状态 |
| `hira issue search <query>` | 关键字搜索 |
| `hira issue runs <id>` | 查看 issue 上智能体跑过的任务 |
| `hira issue rerun <id>` | 重跑最近一次智能体任务 |
| `hira issue comment <id> ...` | 嵌套：看 / 发评论 |
| `hira issue subscriber <id> ...` | 嵌套：订阅 / 取消订阅 |
| `hira project list/get/create/update/delete/status` | Project CRUD |

## 智能体和 Skill

| 命令 | 用途 |
|---|---|
| `hira agent list` | 列出工作区的智能体 |
| `hira agent get <slug>` | 查看智能体配置 |
| `hira agent create ...` | 创建智能体 |
| `hira agent update <slug> ...` | 修改智能体 |
| `hira agent archive <slug>` | 归档 |
| `hira agent restore <slug>` | 恢复归档的智能体 |
| `hira agent tasks <slug>` | 查看智能体的任务历史 |
| `hira agent skills ...` | 嵌套：挂载 / 卸载 Skill |
| `hira skill list/get/create/update/delete` | Skill CRUD |
| `hira skill import ...` | 从 GitHub / ClawHub / 本机导入 Skill |
| `hira skill files ...` | 嵌套：管理 Skill 的文件 |

## Autopilots

| 命令 | 用途 |
|---|---|
| `hira autopilot list` | 列出工作区所有 autopilot |
| `hira autopilot get <id>` | 查看单个 autopilot |
| `hira autopilot create ...` | 创建 autopilot |
| `hira autopilot update <id> ...` | 修改 |
| `hira autopilot delete <id>` | 删除 |
| `hira autopilot runs <id>` | 查看运行历史 |
| `hira autopilot trigger <id>` | 手动触发一次 |

## 守护进程和运行时

| 命令 | 用途 |
|---|---|
| `hira daemon start` | 启动 daemon（默认后台；加 `--foreground` 前台跑）|
| `hira daemon stop` | 停止 daemon |
| `hira daemon restart` | 重启 daemon |
| `hira daemon status` | 查看 daemon 是否在线 + 并发情况 |
| `hira daemon logs` | 查看 daemon 日志 |
| `hira runtime list` | 列出当前工作区的 runtime |
| `hira runtime usage` | 查看资源使用情况 |
| `hira runtime activity` | 近期活动记录 |
| `hira runtime ping <id>` | 立即戳一次 runtime 检查在线 |
| `hira runtime update <id> ...` | 更新 runtime 配置 |

## 杂项

| 命令 | 用途 |
|---|---|
| `hira repo checkout <url>` | 把 repo 拉到本地以供智能体使用 |
| `hira config` | 查看 / 修改 CLI 本地配置 |
| `hira version` | 显示 CLI 版本 |
| `hira update` | 升级 CLI 到最新版 |
| `hira attachment download <id>` | 下载 issue / 评论的附件 |

## 查完整 flag

每条命令都支持 `--help`：

```bash
hira issue create --help
hira agent update --help
```

v2 会给每条命令一个独立的详细 reference 页。
